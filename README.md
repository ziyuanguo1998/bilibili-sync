# Bilibili Sync - B站视频同步器

和异地的 TA 一起同步观看 Bilibili 视频。

## 功能

- **房主/客人身份** —— 房主控制播放进度，客人自动跟随，单向同步不回环
- **视频 ID 识别** —— 自动读取当前 B 站视频的 BV 号（支持分 P、番剧、电影）
- **进度同步** —— 房主跳转进度，客人自动跟随
- **播放/暂停同步** —— 房主暂停或播放，客人同步响应
- **时间偏移量** —— 客人可微调 ±500ms 偏移量，补偿网络延迟差异
- **房间系统** —— 输入相同房间名即可配对，无需注册
- **断线重连** —— 网络中断后自动重新连接并恢复同步

## 架构

```
  房主的浏览器                         服务器                     客人的浏览器
 +------------------+          +---------------+          +------------------+
 | Chrome 扩展      |          |   Node.js     |          | Chrome 扩展      |
 |                  |          |   WebSocket   |          |                  |
 | [Content Script] | -- WS -> |   服务器       |  -- WS ->| [Content Script] |
 | [Service Worker] |          |   (ws 库)     |          | [Service Worker] |
 | [Popup 弹窗]     |          +---------------+          | [Popup 弹窗]     |
 +------------------+                                     +------------------+
       操作发送 -->                 中继转发               --> 接收并应用
```

## 快速开始

### 环境要求

- Node.js >= 18
- Chrome 浏览器（支持 Manifest V3）

### 1. 启动服务器

```bash
cd server
npm install
npm start
```

服务器会在 `ws://localhost:8080` 启动。如需自定义端口：

```bash
PORT=3000 npm start
```

### 2. 安装 Chrome 扩展

1. 打开 Chrome 浏览器，地址栏输入 `chrome://extensions`
2. 右上角打开「开发者模式」
3. 点击「加载已解压的扩展程序」
4. 选择项目中的 `extension/` 目录

### 3. 开始同步

1. 两人都安装好扩展后，打开同一个 B 站视频页面
2. 点击浏览器右上角的扩展图标，打开弹窗
3. 输入一个相同的房间名（任意字符串，如 `我们的房间`）
4. **一方选择「房主」**，**另一方选择「客人」**
5. 两人都点击「加入房间」
6. 房主操作视频时，客人会自动同步！

### 身份说明

| 身份 | 行为 |
|------|------|
| **房主** | 播放/暂停/拖动进度条会同步给房间内所有客人 |
| **客人** | 只接收房主的同步，自己的操作不会影响房主 |

> 客人可在弹窗中使用「时间偏移量」滑块（±500ms），微调和房主之间的进度差异。

## 支持的页面

- `bilibili.com/video/BVxxx` —— 普通视频页面（含分 P）
- `bilibili.com/bangumi/play/epXXX` —— 番剧/电影页面
- `bilibili.com/bangumi/play/ssXXX` —— 番剧（season ID）

## 消息协议

客户端与服务器之间使用 JSON 格式的 WebSocket 消息通信：

| 消息类型 | 方向 | 用途 |
|----------|------|------|
| `JOIN_ROOM` | 客户端 -> 服务器 | 加入同步房间 |
| `LEAVE_ROOM` | 客户端 -> 服务器 | 退出房间 |
| `SEEK` | 房主 -> 服务器 -> 客人 | 跳转到指定时间 |
| `PLAY` | 房主 -> 服务器 -> 客人 | 播放 |
| `PAUSE` | 房主 -> 服务器 -> 客人 | 暂停 |
| `SYNC_STATE` | 房主 -> 服务器 -> 客人 | 完整状态同步（新客人加入时） |
| `HEARTBEAT` | 客户端 -> 服务器 | 心跳保活 |

## 常见问题

### 连接不上服务器？

- 确认服务器已启动（终端应显示 `监听端口: 8080`）
- 如果不在同一局域网，需要将服务器部署到公网，并在扩展设置中修改服务器地址
- 检查防火墙是否放行了对应端口

### 同步有延迟？

- 正常网络环境下延迟在 1 秒以内
- 客人可使用「时间偏移量」滑块微调 ±500ms 来补偿
- 如果服务器部署在远端，延迟与你和服务器之间的网络距离相关

### 视频进度不同步？

- 确认两人观看的是同一个视频（相同的 BV 号或番剧集数）
- 确认一方是「房主」，另一方是「客人」
- 确认弹窗中显示「在线：2 人」
- 尝试退出房间后重新加入

### 切换清晰度后同步失效？

- 系统会自动检测视频元素替换并重新绑定监听器，正常情况下会自动恢复
- 如果仍有问题，刷新页面即可

### 如何部署到公网？

将 `server/` 部署到任意支持 Node.js 的服务器上，然后在扩展设置中将服务器地址改为 `ws://你的服务器IP:端口`。如需加密传输，可使用 `wss://` 配合 Nginx 反向代理实现 TLS。
